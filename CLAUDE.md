# CLAUDE.md — AI-Assisted Development Context

This file provides context for AI coding assistants working on this codebase.

## Project Overview

Mobile Claw is a Capacitor plugin (`capacitor-mobile-claw`) that embeds a full AI agent runtime on Android/iOS via an embedded Node.js worker. The agent uses the Pi framework by Mario Zechner.

## Repository Structure

```
src/                    # TypeScript plugin source → compiled to dist/esm/
  engine.ts             # MobileClawEngine — the core public API class
  definitions.ts        # All TypeScript interfaces and event types
  plugin.ts             # Capacitor WebPlugin wrapper
  services/             # Bridge protocol message types
  mcp/                  # MCP server, transports, and tool types
nodejs-assets/          # Embedded Node.js worker (ships to device)
  nodejs-project/
    main.js             # Worker entry point (agent loop, tools, sessions)
    mcp-agent-tools.js  # Converts MCP tools to agent tools
    mcp-bridge-client.js # Client for device tool bridge
examples/reference-app/ # Vue 3 demo app (private, not published)
test/e2e/               # 13 E2E test files
docs/                   # Extended documentation
```

## Build & Test Commands

```bash
npm install             # Also installs worker deps via postinstall hook
npm run build           # TypeScript → dist/esm/
npm run typecheck       # tsc --noEmit
npm test                # Vitest unit tests
npm run test:e2e        # E2E tests (needs ANTHROPIC_API_KEY)
npm run lint            # Biome check
npm run lint:fix        # Biome auto-fix
```

## Key Architecture

- **Bridge protocol** (`src/services/bridge-protocol.ts`): Typed messages between UI and worker via Capacitor-NodeJS message channel
- **MCP transports**: Bridge (in-process IPC, default) and STOMP (remote WebSocket, opt-in)
- **Pi framework**: `@mariozechner/pi-ai` and `@mariozechner/pi-agent-core` power the agent loop
- **Worker deps**: `nodejs-assets/nodejs-project/node_modules/` is NOT in git — installed via postinstall hook, but IS included in npm package via prepack

## Coding Conventions

- TypeScript strict mode, ES2022 target, ESNext modules
- No semicolons, single quotes, trailing commas
- 2 spaces indentation, 120 char line width
- Biome for linting and formatting
- Conventional commits with scopes: engine, mcp, worker, tools, app, docs, ci, deps

## Files to Never Modify Directly

- `dist/` — generated by `tsc`
- `nodejs-assets/nodejs-project/node_modules/` — committed to git, update via `npm run worker:install`
- `nodejs-assets/nodejs-project/package-lock.json` — only update via `npm install` in that directory

## Important: Worker node_modules are committed to git

The `nodejs-assets/nodejs-project/node_modules/` directory is committed to git intentionally.
The embedded Node.js worker runs on-device where there's no package manager. These deps must
ship as-is in the npm package. Update them with `npm run worker:install` then commit.

## Common Tasks

- **Add a bridge message**: Define types in `src/services/bridge-protocol.ts`, handle in `src/engine.ts`, implement in `nodejs-assets/nodejs-project/main.js`
- **Add an MCP transport**: Implement in `src/mcp/transport/`, register in `transport-manager.ts`
- **Add a device tool**: See `docs/creating-tools.md`
